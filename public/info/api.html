AnimaID API

Comprehensive RESTful API for integrating with AnimaID's data and functionality.

Authentication

JWT-based authentication system with role-based access control.

RESTful Design

Standard HTTP methods with JSON responses for predictable integration.

Security

HTTPS encryption, input validation, and rate limiting.

Base URL

https://your-domain.com/api/v1/

Available Endpoints

/auth

Authentication and user session management

  • POST /auth/login - User login
  • POST /auth/refresh - Refresh access token
  • POST /auth/logout - User logout
  • GET /auth/me - Get current user info

/users

User management (Staff/Admin only)

  • GET /users - List users
  • POST /users - Create user
  • GET /users/{id} - Get user details
  • PUT /users/{id} - Update user
  • DELETE /users/{id} - Delete user

/children

Children and family management

  • GET /children - List children
  • POST /children - Register child
  • GET /children/{id} - Get child details
  • PUT /children/{id} - Update child
  • DELETE /children/{id} - Remove child

/calendar

Activity scheduling and management

  • GET /calendar - Get calendar events
  • POST /calendar - Create event
  • GET /calendar/{id} - Get event details
  • PUT /calendar/{id} - Update event
  • DELETE /calendar/{id} - Delete event

/wiki

Knowledge base and documentation

  • GET /wiki - List wiki pages
  • POST /wiki - Create page
  • GET /wiki/{id} - Get page content
  • PUT /wiki/{id} - Update page
  • DELETE /wiki/{id} - Delete page

Authentication Example

// Login to get access token

curl -X POST /api/auth/login \\

-H "Content-Type: application/json" \\

-d '{"username": "your-user", "password": "your-pass"}'


// Use token in subsequent requests

curl -H "Authorization: Bearer YOUR_TOKEN" \\

Rate Limiting

API requests are rate-limited to prevent abuse. Standard limits:

  • Authenticated requests: 1000 requests per hour
  • Public endpoints: 100 requests per hour per IP
  • Login attempts: 5 attempts per 15 minutes per IP
