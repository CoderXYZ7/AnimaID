API Documentation

RESTful API endpoints for authentication and system management

Overview

The AnimaID API provides RESTful endpoints for authentication, user management, and system administration. All endpoints return JSON responses and use JWT tokens for authentication.

Base URL

https://your-domain.com/api/

Format

JSON (UTF-8)

Authentication

JWT Bearer Tokens

Rate Limits

1000 req/hour (authenticated)

Authentication Endpoints

POST /api/auth/login

Public

Authenticate a user and return JWT token

Request Body:

{
  "username": "string",
  "password": "string"
}

Response (200):

{
  "success": true,
  "token": "jwt_token",
  "user": { ... },
  "expires_at": "2025-11-02T15:29:49Z"
}

POST /api/auth/logout

Authenticated

Invalidate the current session token

GET /api/auth/me

Authenticated

Get current user information and roles

User Management

Endpoint Method Access Description
/api/users GET Admin Get paginated list of users
/api/users POST Admin Create a new user
/api/users/{id} GET Admin Get specific user details
/api/users/{id} PUT Admin Update user information
/api/users/{id} DELETE Admin Deactivate a user

Role Management

Endpoint Method Access Description
/api/roles GET Admin Get all available roles
/api/roles POST Admin Create a new custom role
/api/roles/{id} PUT Admin Update role permissions

System Endpoints

GET /api/system/status

System health check

Admin

POST /api/system/backup

Trigger system backup

Admin

Error Responses

401 Unauthorized

{"error": "Authentication required"}

403 Forbidden

{"error": "Insufficient permissions"}

404 Not Found

{"error": "Resource not found"}

422 Unprocessable Entity

{"error": "Validation failed", "details": {...}}

500 Server Error

{"error": "Internal server error"}

Usage Guidelines

  • • All requests expect application/json content type
  • • Include Authorization: Bearer header for authenticated requests
  • • Handle HTTP status codes properly (200, 201, 401, 403, 404, 422, 500)
  • • Respect rate limits and implement exponential backoff for retries
  • • All dates follow ISO 8601 format (YYYY-MM-DDTHH:mm:ssZ)
